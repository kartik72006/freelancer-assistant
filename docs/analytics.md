# 📊 Product Analytics Module

## Overview

The Product Analytics module provides actionable insights into proposal generation, user behavior, and product performance for the AI Freelancer Proposal Assistant.

Instead of displaying raw database statistics, the analytics dashboard focuses on business metrics that help evaluate product adoption, proposal quality, user engagement, and overall platform health.

---

# Objectives

The analytics system was designed to answer the following questions:

- How many proposals are being generated?
- How many proposals are actually sent?
- What percentage of proposals are accepted?
- Which clients generate the highest revenue?
- Which features are used the most?
- How is proposal quality changing over time?
- What is the overall health of the product?

---

# Product Metrics

## North Star Metric

**Successful Proposals Generated**

A proposal is considered successful when it has been accepted by the client.

This metric directly reflects the value delivered by the product rather than simply measuring activity.

---

## Primary KPIs

| Metric | Description |
|---------|-------------|
| Total Proposals | Total proposals created |
| Acceptance Rate | Accepted proposals / Sent proposals |
| Average AI Score | Average proposal quality score generated by AI review |
| Quoted Revenue | Total value quoted across generated proposals |

# Analytics Events

The application records meaningful user actions using an AnalyticsEvent table.

Events include:

```
proposal_generated
proposal_sent
proposal_accepted
proposal_rejected
proposal_edited
proposal_deleted
proposal_duplicated
proposal_exported
```

Only user-facing actions are tracked.

Internal processing steps such as AI analysis or pricing generation are intentionally excluded to keep analytics meaningful.

---

# Backend Architecture

Analytics follows a layered architecture.

```
Analytics Router
        │
        ▼
Analytics Service
        │
        ▼
Analytics Repository
        │
        ▼
Database
```

This separation keeps business logic independent from database queries.
